PRODUCT REQUIREMENTS DOCUMENT
Ukuran AI
Versi: 1.0
Status: Draft MVP
Kategori: AI Construction Estimation / Quantity Takeoff / RAB
Target pasar: Kontraktor kecil-menengah, mandor, tukang, toko bangunan, dan pemilik rumah di Indonesia
Platform awal: Web responsif
Bahasa: Bahasa Indonesia
________________________________________
1. Ringkasan Produk
Ukuran AI adalah aplikasi berbasis AI yang membantu kontraktor, mandor, dan tukang membuat estimasi kebutuhan material serta Rencana Anggaran Biaya atau RAB dari denah bangunan sederhana.
Pengguna dapat mengunggah foto denah, gambar digital, atau coretan denah 2D. Sistem akan membantu mendeteksi bentuk ruangan, membaca dimensi, menghitung luas dan volume pekerjaan, mengonversinya menjadi kebutuhan material, lalu menyusun estimasi biaya berdasarkan harga material dan upah di wilayah pengguna.
Produk tidak ditujukan untuk menggantikan estimator profesional atau insinyur sipil. Produk berfungsi sebagai asisten perhitungan awal yang mempercepat proses quantity takeoff dan penyusunan RAB.
________________________________________
2. Latar Belakang
Kontraktor kecil, mandor, dan tukang di Indonesia masih banyak menghitung kebutuhan material menggunakan:
•	Catatan manual.
•	Kalkulator.
•	Spreadsheet sederhana.
•	Pengalaman lapangan.
•	Perkiraan berdasarkan proyek sebelumnya.
Proses tersebut memiliki beberapa masalah:
•	Membutuhkan waktu lama.
•	Rawan kesalahan hitung.
•	Sulit memperbarui harga material.
•	Sulit menjelaskan rincian biaya kepada pelanggan.
•	Tidak memiliki format RAB yang konsisten.
•	Sulit menyimpan dan membandingkan estimasi antarproyek.
Software konstruksi profesional yang tersedia umumnya:
•	Berorientasi pada proyek korporasi.
•	Kompleks untuk digunakan.
•	Memerlukan gambar teknis yang rapi.
•	Tidak menggunakan istilah lokal.
•	Tidak memiliki harga material berdasarkan wilayah Indonesia.
•	Terlalu mahal untuk kontraktor kecil.
Ukuran AI akan menyederhanakan proses tersebut melalui pendekatan mobile-friendly, lokal, dan mudah digunakan.

Luas lantai = luas polygon ruangan

Luas kotor dinding = panjang dinding × tinggi dinding

Luas bersih dinding =

15. Kebutuhan Nonfungsional
15.1 Performa
•	Dashboard harus terbuka dalam waktu maksimal 3 detik pada koneksi normal.
•	Upload harus memberikan indikator progres.
•	Pemrosesan denah sederhana ditargetkan selesai kurang dari 60 detik.
•	Editor harus berjalan minimal 30 FPS pada laptop kelas menengah.
•	Perubahan harga harus memperbarui total dalam waktu kurang dari 1 detik.
15.2 Keandalan
•	Data proyek harus tersimpan otomatis.
•	Sistem harus memiliki autosave editor.
•	Sistem harus menyimpan versi terakhir sebelum perubahan besar.
•	Kegagalan AI tidak boleh menghapus input pengguna.
15.3 Keamanan
•	Komunikasi harus menggunakan HTTPS.
•	File pengguna tidak boleh dapat diakses pengguna lain.
•	Akses proyek harus berdasarkan user ID atau team ID.
•	Password tidak disimpan secara plaintext.
•	Link berbagi harus memiliki token unik.
•	Pengguna dapat menghapus file dan proyek.
15.4 Skalabilitas
Arsitektur harus memungkinkan:
•	Pemrosesan AI melalui queue.
•	Penambahan worker.

17. Confidence Score
Sistem harus menggunakan tingkat keyakinan:
Tinggi
Confidence lebih dari atau sama dengan 85%.
•	Elemen ditampilkan normal.
•	Tidak wajib diperiksa, tetapi tetap dapat diedit.
Sedang
Confidence 60–84%.

Rendah
Confidence kurang dari 60%.
•	Elemen diberi tanda merah.
•	Tidak digunakan dalam perhitungan sebelum dikonfirmasi.

22. Formula MVP
Luas lantai
A = luas polygon ruangan
Panjang dinding
L = jumlah panjang seluruh segmen dinding
Luas kotor dinding
A_dinding_kotor = L × tinggi_dinding
Luas bukaan
A_bukaan =
jumlah luas pintu + jumlah luas jendela
Luas bersih dinding
A_dinding_bersih =
A_dinding_kotor - A_bukaan
Waste
quantity_final =
quantity_base × (1 + waste_percentage)
Total item RAB
total_item =
volume × harga_satuan
Harga satuan
harga_satuan =
biaya_material + biaya_upah + biaya_alat
Total akhir
subtotal =
jumlah seluruh total_item

total_akhir =
subtotal
+ overhead
+ profit
+ pajak
- diskon
